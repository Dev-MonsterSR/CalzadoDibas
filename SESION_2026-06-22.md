## 🎯 Features principales entregadas

### 1. Sistema QR de recojo completo

**Backend** (`backend/src/controllers/orderController.js`):
- `GET /api/orders/:id/qr` — genera QR con código JWT
- El QR contiene solo el token puro (no JSON wrapper) para que `html5-qrcode` lo decodifique correctamente
- Response incluye `code: '#000022'` (legible para dictar) + `token` (JWT para escanear)

**Backend** (`backend/src/controllers/inventoryController.js`):
- `POST /api/inventory/verify-qr` — verifica QR o código corto
- **Acepta múltiples formatos**: `#000022`, `000022`, JWT puro, o JSON retrocompatible
- Transición automática: si está en `pagado` o `preparando`, marca como `listo_recojo` → `entregado` en un solo paso
- Bloquea re-entregas con mensaje claro

**Frontend** (`frontend/src/pages/OrderDetail.jsx`):
- Genera QR en `pagado` (no espera `listo_recojo`)
- Código `#000022` clickable, copiable al portapapeles
- Mensaje contextual según estado

**Frontend** (`frontend/src/components/seller/QRScanner.jsx`):
- Modo cámara + modo manual (texto)
- Fix del error "Cannot transition to a new state, already under transition" (race condition al iniciar cámara)
- Validación del estado del scanner antes de `start()`

### 2. Panel del vendedor funcional

**Endpoint** (`backend/src/models/Order.js` `findPickupReadyByLocation`):
- ANTES: solo mostraba órdenes en `listo_recojo`
- AHORA: muestra `pagado`, `preparando`, `listo_recojo` (ordenadas por prioridad)
- Resultado: el vendedor pasó de ver 0 órdenes a ver 16+

### 3. Tailwind v3 local sin bloqueo de ad-blockers

**Problema**: Brave/Cloudflare bloqueaba `cdn.tailwindcss.com` con `ERR_BLOCKED_BY_CLIENT`.

**Solución**:
- Bundle de Tailwind v3.4.17 descargado en `frontend/public/tailwind-cdn.js` (418KB)
- `index.html` usa `<script src="/tailwind-cdn.js" defer>` (mismo dominio, no bloqueado)
- `window.tailwind.suppressProductionWarning = true` para evitar el warning de "should not be used in production"
- **Orden de scripts crítico**: namespace → suprimir warning → cargar bundle → aplicar config

**Bug que rompía todo**: el `tailwind.config = {...}` corría antes de que `window.tailwind` existiera → `Uncaught ReferenceError: tailwind is not defined`. Solución: pre-crear `window.tailwind = {}` en un script previo.
